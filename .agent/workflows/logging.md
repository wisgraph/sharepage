---
description: 매일 진행한 작업과 계획을 문서화하여 기록하는 워크플로우
---

1. **날짜 확인**: 
   - 오늘 날짜(YYYY-MM-DD)를 확인합니다. `date` 명령어를 사용하지 말고 시스템 시간을 참고하세요.

2. **저널 파일 확인**:
   - `.agent/journals/{today}.md` 파일이 존재하는지 확인합니다.
   - 예: `.agent/journals/2026-02-02.md`

3. **기존 내용 읽기 (CRITICAL)**:
   - 파일이 존재하면 `view_file` 도구를 사용하여 **반드시** 기존 내용을 먼저 읽습니다.
   - ⚠️ **주의**: 기존 내용을 읽지 않고 `write_to_file`을 사용하면 **절대 안 됩니다**. 이전 기록이 삭제될 수 있습니다.

4. **내용 작성 및 병합**:
   - 읽어온 기존 내용 뒤에 새로운 작업 내역을 추가합니다.
   - 만약 파일이 없었다면, 새로운 파일을 생성합니다.

5. **형식 준수**:
   - 아래 형식을 따릅니다.
   ```markdown
   ## 🕒 {Time} 작업 기록
   - **[{Type}] {Title}**:
       - {Detail 1}
       - {Detail 2}
   ```
   - Type 예시: Feature, Bug, Refactor, Chore, Doc

6. **저장**:
   - 파일이 있었으면: `replace_file_content`로 기존 내용 유지하며 추가. (또는 전체 내용을 합쳐서 `write_to_file` 하되, 기존 내용 유실 주의)
   - 파일이 없었으면: `write_to_file`로 생성.
