# 📑 클로드코드 직접 만든 사람이 공개한 사용법, 저도 충격받았습니다

![](http://www.youtube.com/watch?v=2WJzwwvzbBQ)

> [!ABSTRACT] 3줄 요약
>
> 1. 클로드 코드를 단일 창에서 수동적으로 쓰지 말고, 독립된 여러 창과 알림 기능을 활용해 오케스트라처럼 다중 에이전트를 지휘해야 한다.

## 1. 클로드 코드를 지휘하는 방법 다중 창 오케스트레이션

> _클로드 코드를 단일 창으로 쓰는 것은 성능의 10%만 활용하는 것이며, 여러 개의 독립된 창을 열어 각각의 역할을 부여하고 통제하는 방식이 필수적이다._

- 다중 창 동시 작업: 화면 창을 1번부터 5번까지 열어 클로드 코드를 각각 실행시키고, 서로 다른 독립된 작업을 동시에 지시한다 (00:30).
  - **오케스트레이션**: 기획, 디자인, 코드 정리 등 여러 역할을 5명의 직원에게 분배하듯 각자의 영역에서만 일하게 하여 맥락이 엉키는 것을 방지하는 작업 조율 방식.
- 알림 기반의 작업 전환: 여러 클로드가 작업을 수행 중일 때 어느 것이 완료되었는지 파악하기 위해 알림을 설정한다 (00:01:08).
  - 대기 상태에 돌입하면 알림이 오고, 사용자는 해당 창으로 넘어가 다음 지시를 내리는 형태로 작업의 연속성을 극대화한다.

## 2. 작업 품질을 3배 높이는 핵심 사이클 계획과 검증

> _무작정 코드를 작성하게 하는 대신 전체 구조를 먼저 설계하게 하고, 완성된 결과물을 AI가 직접 실행하여 확인하도록 만들어 오류를 사전에 차단한다._

> [!QUOTE] 집 짓기 비유
> "계획에 에너지를 쏟아라. 그래야 클로드가 작업을 한 번에 끝낸다."

- 계획 우선(Plan Mode)의 중요성: 클로드가 실제로 코드를 건드리기 전에 상황을 분석하고 계획만 선제적으로 짜도록 한다 (00:01:36).
- 자가 검증(Self-Verification) 루틴: 클로드에게 작업 완료 후 스스로 결과물이 맞는지 점검할 수 있는 수단과 방법을 제공한다 (00:02:08).
  - 간단한 작업은 코드를 직접 실행해보게 하고, 복잡한 작업은 브라우저를 열어 UI 렌더링을 확인하게 한다.
  - **요리사 비유**: "김치찌개를 끓여달라고 한 뒤 요리사가 스스로 맛을 보지 않으면 결과물의 품질을 담보할 수 없다"는 원리. 확인 방법을 주면 결과물 품질이 2~3배 향상된다.
- 최고 성능 모델 활용: 싸고 빠른 모델로 여러 번 수정하는 것보다, 느리지만 가장 똑똑한 모델을 쓰는 것이 수정 횟수를 줄여주어 결과적으로 훨씬 빠르다 (00:03:10).

## 3. 시스템 맞춤화 복리 학습 메모장과 단축 명령어

> _AI의 빈출 오류나 반복되는 작업 패턴을 전용 메모장과 단축 명령어로 규격화하여, 점진적으로 내 작업 방식에 최적화된 시스템을 만든다._

- 클로드 전용 메모장 운영: 프로젝트 폴더 내에 팀의 규칙, 방향, 자주 틀리는 실수 목록을 담은 파일을 별도로 생성한다 (00:03:27).
  - **복리 학습**: 에러 발생 시 "이 메모장을 업데이트해서 같은 실수 다시 하지 마"라고 지시하면, 새 직원이 매뉴얼을 익히듯 실수를 줄여나가는 누적된 학습 효과를 얻는다.
- 단축 명령어 및 분업화: '저장 후 공유 및 검토 요청'처럼 자주 하는 작업 순서를 하나의 단축어로 묶어 스마트폰 단축어처럼 활용한다 (00:04:06).
  - 오류 찾기, 규칙 확인, 이력 파악 등을 여러 클로드에게 분업 시켜 사람이 한 번 검토할 때 이미 다각도의 확인이 병렬로 완료되게 한다.

## 4. 실전 팁 1 맥락 유지와 도메인 영역 분리

> _새로운 세션을 시작할 때 발생하는 AI의 '기억 상실' 문제를 해결하고, 프로젝트 규모 확장에 따른 맥락 혼선을 폴더 단위 격리로 방지한다._

- 대화 내용 파일화 (Context Storage): 클로드는 새로 시작하면 이전 대화를 매번 처음 보는 사람처럼 완전히 잊어버리므로, 진행 상황과 요약된 내용을 파일로 저장해둔다 (00:05:08).
  - 새 직원을 위한 인수인계 문서처럼 세션 시작 시 해당 요약 파일을 주입하면 매번 처음부터 설명하는 시간을 획기적으로 절약할 수 있다.
- 프로젝트 영역 분리: 프로젝트 규모가 커지면 결제, 사용자, 관리자 등 기능별로 폴더를 나눈다 (00:05:44).
  - **도메인 분리**: 각 영역(폴더)마다 전담 클로드를 하나씩 배치하여 서로의 영역을 침범하지 않고 자신이 맡은 도메인에만 집중하게 만들어 치명적인 실수를 방지하는 기법. 인사팀이 재무팀 일에 끼어들지 않는 것과 같다.

## 5. 실전 팁 2 설계 주도 개발과 나만의 스킬화

> _작업 전 설계도를 강제하고 커스텀 스크립트를 작성하여 AI 보조 코딩의 효율성과 정확도를 극대화한다._

- 설계 주도 개발 도입: 무작정 기능 추가를 지시하기 전에 설계서부터 작성한다 (00:06:12).
  - **SDD (Spec Driven Development)**: 코드를 짜기 전 무엇을 만들지, 어떻게 생겼고 동작할지를 명확한 문서(스펙)로 먼저 정의하는 개발 방식.
  - 설계서가 존재하면 클로드의 작업 방향이 흔들리지 않아 수십 번의 무의미한 코드 수정을 원천적으로 차단해 준다.
- 작업 완료 알림 (Hook) 설정: 클로드가 작업 중일 때 화면을 계속 쳐다보며 대기하는 비효율을 없애기 위해 작업 완료 소리 알림을 설정한다 (00:06:43).
- 나만의 단축 매뉴얼(Skill) 구축: 단순 명령어를 넘어서 작업 순서, 판단 기준, 빈출 실수까지 적어둔 종합 가이드 파일을 만든다 (00:07:19).
  - "썸네일 만들어 줘" 한 마디에 해당 스킬 파일을 참조하여 처음부터 끝까지 알아서 처리하게 만드는 강력한 맞춤형 환경을 의미한다.
