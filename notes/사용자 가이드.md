# 🚀 SharePage 사용자 가이드

SharePage는 옵시디언(Obsidian) 문서를 가장 아름답고 스마트하게 공유할 수 있는 플랫폼입니다. 이제 새로운 폴더 구조를 통해 더욱 깔끔하게 프로젝트를 관리할 수 있습니다.

## 1. 문서 게시 및 구조

SharePage는 파일의 성격에 따라 폴더를 구분하여 관리합니다.

- **`notes/`**: 작성한 모든 마크다운(`.md`) 원본 파일을 이 폴더에 넣습니다.
- **`images/`**: 문서에 사용될 이미지 파일들을 이 폴더에 보관합니다.
- **`posts/`**: 배포 시 검색 엔진(SEO)과 SNS 미리보기를 위해 자동으로 생성되는 HTML 파일들이 저장됩니다.
- **업로드**: 파일을 각 폴더에 넣고 `gh-pages` 브랜치에 Push하면 배포가 완료됩니다.

## 2. 문서 작성 및 메타데이터 (YAML)

문서 최상단에 YAML Frontmatter를 추가하여 페이지를 설정할 수 있습니다.

```yaml
---
title: 문서 제목 (미출력 시 파일명 사용)
description: 미리보기에 표시될 요약 문장
thumbnail: cover.jpg # (선택) 목록 및 공유 시 표시될 썸네일 이미지 (images/ 폴더 내 파일명)
tags:
  - "#태그1"
  - "#태그2"
---
```

## 3. 오픈그래프(OG) 및 미리보기 최적화

공유 시 나타나는 썸네일 이미지는 다음 우선순위로 결정됩니다.

1.  **YAML Frontmatter의 `thumbnail` 필드 (최우선)**
2.  본문에 삽입된 첫 번째 유튜브 영상의 썸네일 (자동 추출)
3.  본문에 삽입된 첫 번째 이미지 (`images/` 폴더 내 이미지)
4.  이미지가 없을 경우: 제목 기반의 **랜덤 색상 그라데이션** 자동 생성

## 4. 유튜브(YouTube) 처리 방식

- **자동 임베드**: 본문에 유튜브 링크를 적으면 자동으로 반응형 플레이어로 변환됩니다.
- **스마트 썸네일**: 영상 링크만 있으면 해당 영상의 고화질 썸네일을 미리보기(OG) 이미지로 사용합니다.

## 5. 이미지 사용 방법

- **옵시디언 방식**: `![[이미지명.jpg]]` 문법을 그대로 사용하세요. SharePage가 자동으로 `images/` 폴더에서 해당 파일을 찾아 연결합니다.
- **주의사항**: 순수하게 원본 파일명 그대로 `images/` 폴더에 넣으시면 됩니다.

---

### 💡 팁
- **정적 HTML 생성**: `scripts/sync.js`가 `notes/`에서 마크다운을 읽어 `posts/`에 최적화된 HTML을 만들어냅니다.
- **다크 모드**: 우측 상단의 테마 스위치를 통해 독자가 원하는 테마로 문서를 읽을 수 있습니다.
