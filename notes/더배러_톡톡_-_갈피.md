# **갈피**는 책 너머 다양한 PC 환경에 맞춰 **책갈피** 역할을 해주길 바랐습니다.

책을 볼 때 어디까지 봤는지, 어디서 다시 읽어야 할지 찾아보신적 있으신가요?
지난번에 어디까지 읽었는지 손끝에 남아있는 두께에 대한 감각을 통해
무작위 페이지를 펴고 읽고, "아 여긴 기억나" 하며 몇페이지를 넘기고,
"여기도 기억나", "여긴 모르겟네", "이건 본적없어"
하며 기억을 더듬어가며, 마지막에 읽은 페이지를 찾아가 보신적 있으실 겁니다.

마지막에 읽었던 페이지를 그 문장을 그때 그때 찾는 일은 꽤 번거로운 일입니다.

하지만 **책갈피**를 이용하면 어디서 다시 읽어야할지 기억을 더듬어가며 찾을 필요가 없습니다.
어제 읽은 그곳에 **책갈피**가 꽂혀있기 때문에 그저 그곳을 펼치기만 하면됩니다.

**갈피**는 책 너머 다양한 PC 환경에 맞춰 **책갈피** 역할을 해주길 바랐습니다.

PC는 셀수 없이 많은 문서와 웹들을 처리합니다. 수만권의 책이 하나의 PC에 들어와 있죠.
우리는 늘, 어제, 저번주, 지난달에 진행하던 작업을
마지막 읽은 책의 페이지를 찾아내듯, 탐색하는 과정을 겪습니다.
그 과정이 순탄할 때도 있지만
오래된 내용일 수록 찾아내지 못하는 상황을 늘 마주합니다.

갈피는 이 문제를 해결합니다.
프로젝트 이름으로 문서들을 묶어 내어 관리합니다.
그리고
**그때 그 환경을 한번에 복구**합니다.

# 1. 문서링크의 저장과 로드

- 갈피 앱은 현재 포커스 된 앱 화면의 정보를 읽어 다시 불러올 수 있는 형태로 가공하여 저장합니다.
  - ![[img.갈피 캡쳐.png]]
- 캡쳐된 앱과 문서정보에 이름을 수정하고, 태그를 입력 할 수 있습니다.
  - 이름과 태그의 수정은 카드를 우클릭 후 `Edit Title`버튼을 눌러 수정이 가능합니다
    또 화면에 보이듯이 `cmd+e` 키로 메뉴접근 없이 빠르게 전환이 가능합니다.
    ![[img.컨텍스트 메뉴.png]]
    ![[img.이름 수정.png]]
  - 태그 수정 시에는 태그 자동완성기능이 동작합니다.
    위 아래 키로 셀렉트하고 엔터를 누르면 자동입력됩니다.
    태그 수정 모드는 이름변경시에 `#` 을 입력하면 자동진행되며,
    태그가 완성되고 스페이스로 공백을 추가하면 태그수정모드는 종료됩니다.
    ![[img.태그 수정.png]] ![[img.이름 수정.png]]
- 저장된 갈피에 커서를 올려놓고 엔터를 누르면 해당 링크가 앱과 함께 열립니다!
  - ![[img.앱 열기.png]]

# 3. 컨텍스트 태그의 활용 - 작업 내역 복구하기

- **태그**의 활용
  - 옵시디언 사용자라면 태그를 정말 잘 활용하실텐데요
  - 처음 사용자라면, 태그를 **적게 사용하기**를 추천하고 있습니다.
  - 태그를 많이 사용하면, 기억상으로 태그와 문서의 연결이 약해지고,
  - 태그 검색시에 문서가 있는지 없는지 확정할 수 없는 상태에 빠지게 됩니다
  - 찾는 문서가 있는지 없는지 불확정적인 상태는
  - 우리가 **눈으로 직접 확인해야만 하는 상황**이 필연적이며
  - 탐색에 N가지 불확정적인 절차가 추가됩니다
  - 이를 제어하기위해 태그 사용을 절제하고,
  - 핵심이 될 **컨텍스트 태그**만 사용하기를 권장하고 있습니다.
- **컨텍스트 태그**란?
  - 갈피 앱은 **타입드태그**를 사용합니다.
    - **타입드태그**란 #type/tagname 꼴의 의미론적 태그 사용을 말하며, 태그 이름에 카테고리를 부여하는 방식으로 사용합니다.
  - 컨텍스트태그를 사용하면, 해당 태그를 갖고있는 문서를 캡쳐할 때
    앱에서 자동으로 해당 컨텍스트 태그를 검색어로 입력처리합니다.
    이를 통해 해당 컨텍스트 태그를 갖고있는 모든 링크를 확인할 수 있으며
    해당 작업에서 사용되는 문서들을 **직접 검색하지 않고도** 탐색할 수 있습니다.
    ![[img.컨텍스트 태그 자동검색.png]]
  - 갈피앱은 #project/name 을 컨텍스트 태그로 기본사용합니다.
    - `설정 > 태그` 섹션에서 변경 및 추가 가능합니다
      모든 링크는 여러개의 태그를 부여할 수 있으며,
      링크이름에 태그를 추가하면 자동분류됩니다.
      ![[img.컨텍스트 태그의 추가 - 설정>태그.png.png]]
  - 컨텍스트 타입으로 특정 작업의 이름을 명시합니다.
    ![[img.컨텍스트 태그 저장.png]]
  - 다시 저장된 문서를 캡쳐하면, 자동검색이 진행됩니다.
    ![[img.자동검색결과.png]]
- **스탬프 태그**
  - 실수로 저장은 했는데, 태그를 안적었다면?, 우측 상단에 **스탬프**를 사용할 수 있습니다.
    먼저 검색어를 제거하고, 추가할 링크를 찾아줍니다.
    그리고 태그자리에 마우스를 올리면,
    직접 태그를 입력하지 않아도
    클릭한번에 스탬프로 태그를 찍을 수 있습니다.
    ![[img.스탬프 태그 사용.png]]
- 이렇게 컨텍스트 태그로 관리되는 링크들은
  - 단축키 `option + cmd + c` 를 통해 일괄 링크복사를 할 수 있고,
  - 단축키 `option + enter` 를 통해 일괄 실행 할 수 있습니다.
