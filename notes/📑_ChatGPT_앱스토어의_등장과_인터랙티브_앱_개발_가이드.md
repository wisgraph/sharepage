## 새로운 골드러시: ChatGPT 앱스토어 생태계

> _단순한 텍스트 챗봇을 넘어, 인터랙티브한 UI를 갖춘 앱 생태계가 조용히 열리고 있다._

현재 많은 사람과 기업들이 ChatGPT 내부에서 앱을 구축할 수 있다는 사실을 알지 못한다. 하지만 이미 Photoshop, Canva 등 거대 기업들은 이 플랫폼에 앱을 출시하고 있다.

- **범용성과 확장성**:
  - 이 앱들은 특정 플랫폼에 종속되지 않는다. 기반 기술인 MCP Apps: 모델 컨텍스트 프로토콜(MCP)의 개방형 확장판으로, AI가 텍스트를 넘어 인터랙티브한 위젯을 렌더링할 수 있게 해주는 개방형 표준을 사용하기 때문이다.
  - Anthropic의 Claude(Interactive Tools), VS Code 등이 이미 이를 도입했으며, 향후 Google 등 다른 플랫폼이 도입하면 개발한 앱은 수정 없이 그대로 작동하게 된다.

> [!QUOTE] 생태계 선점의 중요성
> "Apple 앱스토어 초기나 Shopify 앱스토어 초기에 진입했던 개발자들이 큰 부를 이뤘듯, 수억 명의 사용자를 가진 ChatGPT 앱스토어의 창은 지금 열려 있으며 영원히 열려 있지는 않을 것입니다."

## ChatGPT 앱의 핵심 아키텍처

ChatGPT 앱의 본질은 약간의 추가 설정이 가미된 **MCP Server**: AI 모델이 외부의 데이터베이스, 파일, 내부 API 등에 안전하게 접근하고 도구를 실행할 수 있게 해주는 중계 서버에 불과하다. 이 서버가 텍스트 대신 시각적 UI를 반환하도록 만드는 것이 핵심이다.

1. **위젯 (Widget)**: 위젯(Widget): ChatGPT 대화창 내부에 iframe 형태로 렌더링되는 React 기반의 미니 애플리케이션이다.
2. **리소스 (Resource)**: 서버 측에서 위젯의 HTML/JavaScript/CSS 코드를 담고 있는 파일 묶음이다.
3. **구조화된 콘텐츠 (Structured Content)**: 서버가 텍스트 대신 위젯(React 앱)으로 직접 보내는 순수한 데이터 페이로드(Data Payload)다.

## 단계별 구축 가이드: 주식 가격 앱 만들기

> _사용자가 주식 가격을 물어보면, 단순 텍스트가 아닌 실시간 차트와 '관심 종목 추가' 버튼이 있는 UI 카드를 렌더링하는 앱의 작동 흐름._

### 1. 서버 설정 및 리소스 등록

가장 먼저 서버가 ChatGPT에게 "나에게 렌더링할 위젯(HTML)이 있다"고 알려야 한다.

- `registerAppResource` 함수를 사용해 위젯의 HTML을 반환하는 고유한 URI를 등록한다.
- 이후 주식 가격을 가져오는 도구(Tool)를 등록할 때, 두 가지 핵심 설정을 추가한다.
  - `meta.ui.resource_uri`: 도구 실행 완료 후 텍스트 대신 **앞서 등록한 위젯을 화면에 띄우도록** 지시한다.
  - `structuredContent`: 주식 가격, 종목명 등의 순수 데이터를 텍스트가 아닌 위젯 프론트엔드로 바로 전송한다.

### 2. 프론트엔드 위젯 렌더링 (React)

서버에서 데이터를 보냈다면, 위젯은 이를 받아 화면에 그려야 한다.

- 위젯 내부에서는 `useApp`이라는 특수한 훅(Hook)을 사용하여 ChatGPT의 iframe 환경과 통신한다.
- `onToolResult` 콜백 함수를 리스닝하고 있다가, 서버에서 도구 실행 결과(구조화된 데이터)가 도착하면 이를 가로채서 매력적인 React 컴포넌트(주식 카드)로 렌더링한다.

### 3. 양방향 상호작용 (Interactivity 구현)

단순히 보여주는 것을 넘어, 위젯 안의 버튼을 눌렀을 때 서버의 기능이 실행되도록 만들어야 진정한 앱이다.

- 위젯에 있는 '관심 종목 추가(Add to Watchlist)' 버튼을 클릭하면 이벤트가 발생한다.
- **가시성 권한(Visibility) 변경**: 기본적으로 도구는 AI 모델(Model)만 호출할 수 있다. 하지만 도구의 가시성을 `app`으로 설정하면, **위젯이 AI를 거치지 않고 서버의 도구를 직접 호출**할 수 있게 된다.
- 클릭 한 번으로 위젯 -> ChatGPT -> 서버로 신호가 가고, 데이터베이스에 종목이 저장된다.

## 고급 기능과 미래 확장성

간단한 정보 조회용 카드를 넘어, ChatGPT 앱은 데스크톱이나 모바일 앱에 준하는 강력한 기능들을 제공할 수 있다.

- **다양한 UI 형태**: 전체 화면(Full Screen) 전환으로 복잡한 워크플로우를 처리하거나, 대화를 나누는 동안 화면 구석에 떠 있는 PIP(Picture-in-Picture) 형태로 게임을 띄워놓을 수도 있다.
- **복합 기능**: 캐러셀(Carousel), 지도(Maps), 미디어 앨범을 렌더링할 수 있다.
- **수익화**: 대화창 내부에서 이탈하지 않고 즉각적으로 결제를 처리하는 완전한 E-commerce 스토어 구축도 가능하다.

## 결론 및 액션 아이템

> _텍스트 프롬프팅의 시대에서, 시각적이고 직관적인 'AI 임베디드 앱'의 시대로 진화하고 있다._

- 핵심 시사점: ChatGPT 앱은 단순히 프롬프트 결과를 예쁘게 포장하는 것이 아니다. 사용자 경험(UX)을 극대화하기 위해 AI의 추론 능력과 현대적인 웹 프론트엔드 기술(React)을 매끄럽게 결합하는 새로운 소프트웨어 패러다임이다.

- Action Items:
  - [ ] **기반 지식 학습**: MCP (Model Context Protocol)의 기본 작동 원리(Server-Tool-Client)를 문서나 이전 튜토리얼을 통해 학습한다.
  - [ ] **아이디어 기획**: 기존에 텍스트로만 제공되던 서비스 중, 버튼이나 시각적 차트가 결합되었을 때 사용자 경험이 폭발적으로 좋아질 수 있는 비즈니스 아이템을 1~2개 구상해본다.
  - [ ] (Thinking): 내 비즈니스나 서비스를 ChatGPT 외부의 독립된 앱으로 유지할 것인가, 아니면 사용자가 이미 머물고 있는 AI 플랫폼 내부의 '플러그인 앱'으로 침투시킬 것인가?
